## 🔧 ধাপ ১: ওয়্যারিং / সংযোগ করা

### ⚡ পাওয়ার সার্কিট

```
ব্যাটারি (7.4V) → বাক কনভার্টার (আউটপুট 5V) → ESP32 VIN + PCA9685 V+
```

> [!CAUTION]
> বাক কনভার্টারের আউটপুট **অবশ্যই 5V এ সেট করুন** মোটর বা ESP32 সংযোগ করার আগে। মাল্টিমিটার দিয়ে চেক করুন!

### 📌 PCA9685 → ESP32 সংযোগ

| PCA9685 পিন | ESP32 পিন | বিবরণ |
|:---:|:---:|---|
| SDA | GPIO 21 | I2C ডেটা লাইন |
| SCL | GPIO 22 | I2C ক্লক লাইন |
| VCC | 3.3V | PCA9685 লজিক পাওয়ার |
| GND | GND | কমন গ্রাউন্ড |
| V+ | 5V (বাক কনভার্টার) | সার্ভো মোটর পাওয়ার |

### 📌 OLED ডিসপ্লে → ESP32 সংযোগ

| OLED পিন | ESP32 পিন | বিবরণ |
|:---:|:---:|---|
| SDA | GPIO 21 | I2C ডেটা (PCA9685 এর সাথে শেয়ার) |
| SCL | GPIO 22 | I2C ক্লক (PCA9685 এর সাথে শেয়ার) |
| VCC | 3.3V | পাওয়ার |
| GND | GND | গ্রাউন্ড |

> [!IMPORTANT]
> PCA9685 এবং OLED উভয়ই **একই I2C বাসে** (GPIO 21/22) থাকবে। তাদের I2C অ্যাড্রেস আলাদা তাই কোনো সমস্যা হবে না। 
> - PCA9685 অ্যাড্রেস: `0x40`
> - OLED অ্যাড্রেস: `0x3C`

### 📌 সার্ভো মোটর → PCA9685 সংযোগ

| PCA9685 চ্যানেল | মোটর | পজিশন |
|:---:|:---:|---|
| Channel 0 | Motor 0 (R1) | ডান সামনের ঊরু |
| Channel 1 | Motor 1 (R2) | ডান পেছনের ঊরু |
| Channel 2 | Motor 2 (L1) | বাম সামনের ঊরু |
| Channel 3 | Motor 3 (L2) | বাম পেছনের ঊরু |
| Channel 4 | Motor 4 (R4) | ডান পেছনের পা |
| Channel 5 | Motor 5 (R3) | ডান সামনের পা |
| Channel 6 | Motor 6 (L3) | বাম সামনের পা |
| Channel 7 | Motor 7 (L4) | বাম পেছনের পা |

### 🔌 ওয়্যারিং ডায়াগ্রাম

```
                    ┌──────────────┐
  ব্যাটারি 7.4V ──→│ বাক কনভার্টার │──→ 5V আউটপুট
                    └──────┬───────┘
                           │
                    ┌──────┴───────┐
                    │              │
              ┌─────┴─────┐  ┌────┴────┐
              │   ESP32   │  │ PCA9685 │
              │           │  │  V+ =5V │
              │ GPIO21=SDA├──┤ SDA     │
              │ GPIO22=SCL├──┤ SCL     │
              │ 3.3V──────├──┤ VCC     │
              │ GND───────├──┤ GND     │
              └─────┬─────┘  └────┬────┘
                    │              │
              ┌─────┴─────┐   ┌───┴───────────┐
              │   OLED    │   │ সার্ভো মোটর   │
              │ SSD1306   │   │ Ch0-Ch7 (8টি) │
              │ 0x3C      │   │ MG90S         │
              └───────────┘   └───────────────┘
```

## 💻 ধাপ ২: Arduino IDE সেটআপ

### ২.১ Arduino IDE ইনস্টল করুন

1. [Arduino IDE ডাউনলোড](https://www.arduino.cc/en/software) করুন (ভার্সন 2.0+)
2. ইনস্টল করে ওপেন করুন

### ২.২ ESP32 বোর্ড সাপোর্ট যোগ করুন

1. Arduino IDE ওপেন করুন
2. **File → Preferences** এ যান
3. "Additional Board Manager URLs" ফিল্ডে পেস্ট করুন:
   ```
   https://raw.githubusercontent.com/espressif/arduino-esp32/gh-pages/package_esp32_index.json
   ```
4. **OK** ক্লিক করুন
5. **Tools → Board → Boards Manager** এ যান
6. "ESP32" সার্চ করুন
7. "**ESP32 by Espressif Systems**" ইনস্টল করুন (v2.0.0 বা তার উপরে)

### ২.৩ প্রয়োজনীয় লাইব্রেরি ইনস্টল করুন

**Tools → Manage Libraries** (Library Manager) এ যান এবং নিচের লাইব্রেরিগুলো ইনস্টল করুন:

| লাইব্রেরি | ভার্সন | কাজ |
|---|---|---|
| **Adafruit PWM Servo Driver Library** | সর্বশেষ | PCA9685 সার্ভো ড্রাইভার কন্ট্রোল |
| **Adafruit SSD1306** | সর্বশেষ | OLED ডিসপ্লে |
| **Adafruit GFX Library** | সর্বশেষ | গ্রাফিক্স রেন্ডারিং |

> [!NOTE]
> আপনার রোবটে PCA9685 আছে, তাই **ESP32Servo** লাইব্রেরি লাগবে না। এর বদলে **Adafruit PWM Servo Driver** ব্যবহার হবে।

## 📥 ধাপ ৩: ফার্মওয়্যার ফ্ল্যাশ করা (কোড আপলোড)

### ৩.১ কোড ফাইল প্রস্তুত করুন

আপনার কম্পিউটারে `spidy-robot/firmware/` ফোল্ডারে নিচের ফাইলগুলো আছে:

```
firmware/
├── sesame-firmware-main.ino     ← মূল কোড (এটি মডিফাই করতে হবে)
├── face-bitmaps.h               ← মুখের ছবি/বিটম্যাপ
├── movement-sequences.h         ← হাঁটা/নাচ/হাত নাড়া ইত্যাদি
└── captive-portal.h             ← ওয়েব কন্ট্রোলার HTML
```

### ৩.২ PCA9685 এর জন্য কোড পরিবর্তন

**সুখবর!** স্পাইডি রোবটের কোডটি ইতিমধ্যেই **PCA9685** এবং **OLED ডিসপ্লে (I2C Pin 21, 22)** এর জন্য আগে থেকেই কনফিগার করা আছে। তাই আপনাকে ম্যানুয়ালি কোনো কোড পরিবর্তন করতে হবে না। আপনি সরাসরি কোড আপলোড করতে পারেন।

### ৩.৩ ESP32 তে কোড আপলোড করুন

1. **ESP32 কে USB কেবল দিয়ে কম্পিউটারে সংযোগ করুন**

2. **Arduino IDE তে বোর্ড সিলেক্ট করুন:**
   - **Tools → Board** → "**ESP32 Dev Module**" সিলেক্ট করুন
   - **Upload Speed**: 921600
   - **Flash Mode**: QIO
   - **Partition Scheme**: Default 4MB with spiffs

3. **পোর্ট সিলেক্ট করুন:**
   - **Tools → Port** → আপনার ESP32 এর COM পোর্ট সিলেক্ট করুন
   - (macOS তে `/dev/cu.usbserial-XXXX` বা `/dev/cu.SLAB_USBtoUART`)

4. **আপলোড করুন:**
   - **Upload বাটন (→)** ক্লিক করুন
   - কম্পাইল ও আপলোড শেষ হওয়ার জন্য অপেক্ষা করুন
   - "Done uploading" দেখলে সফলভাবে আপলোড হয়েছে ✅

> [!WARNING]
> আপলোড ফেইল হলে:
> - ESP32 এর **BOOT** বাটন চেপে ধরে আবার আপলোড করুন
> - ভিন্ন USB কেবল ট্রাই করুন
> - USB ড্রাইভার ইনস্টল আছে কিনা চেক করুন (CP210x বা CH340)

## 📡 ধাপ ৪: রোবট চালু করা ও সংযোগ

### ৪.১ প্রথমবার চালু করা

1. **সব ওয়্যারিং চেক করুন** — কোনো শর্ট সার্কিট নেই তা নিশ্চিত করুন
2. **ব্যাটারি সংযোগ করুন** বা USB কেবলে রাখুন
3. **পাওয়ার অন করুন**
4. OLED ডিসপ্লেতে মুখ দেখা যাবে 😊
5. ESP32 একটি **WiFi নেটওয়ার্ক তৈরি করবে**

### ৪.২ WiFi দিয়ে সংযোগ

| সেটিং | মান |
|---|---|
| **WiFi নাম (SSID)** | `Sesame-Controller` |
| **পাসওয়ার্ড** | `12345678` |
| **IP অ্যাড্রেস** | `192.168.4.1` |

**ধাপগুলো:**

1. আপনার **মোবাইল ফোন** বা **ল্যাপটপ** থেকে WiFi সেটিংসে যান
2. `Sesame-Controller` নেটওয়ার্ক খুঁজুন
3. পাসওয়ার্ড `12345678` দিয়ে সংযোগ করুন
4. মোবাইলে একটি ক্যাপটিভ পোর্টাল অটো-ওপেন হবে
5. না হলে ব্রাউজারে `http://192.168.4.1` লিখুন

### ৪.৩ ওয়েব কন্ট্রোলার (আমাদের কাস্টম)

আমরা একটি আলাদা **স্পাইডার থিম ওয়েব কন্ট্রোলার** তৈরি করেছি।  
এটি আপনার কম্পিউটারে রান করতে:

```bash
cd spidy-robot/software/web-controller
python3 -m http.server 8080
```

তারপর ব্রাউজারে `http://localhost:8080` খুলুন।

**রোবটের সাথে সংযোগ করতে:**
1. ওয়েব কন্ট্রোলারে "Demo" পিলে ক্লিক করুন
2. Robot IP ফিল্ডে `192.168.4.1` লিখুন
3. "Connect" বাটনে ক্লিক করুন

## 🔍 ধাপ ৬: সমস্যা সমাধান (Troubleshooting)

### ❌ আপলোড ফেইল হচ্ছে

- ESP32 এর **BOOT** বাটন চেপে ধরুন আপলোডের সময়
- অন্য USB কেবল ব্যবহার করুন
- **Tools → Port** এ সঠিক পোর্ট সিলেক্ট করুন
- CP210x বা CH340 USB ড্রাইভার ইনস্টল আছে কিনা দেখুন

### ❌ OLED ডিসপ্লে কাজ করছে না

- SDA/SCL ওয়্যার সঠিক পিনে আছে কিনা চেক করুন (GPIO 21/22)
- ওয়্যার ঠিকমতো সোল্ডার করা আছে কিনা দেখুন
- Serial Monitor (115200 baud) এ "SSD1306 allocation failed" দেখলে I2C সমস্যা

### ❌ সার্ভো মোটর নড়ছে না

- PCA9685 তে V+ পাওয়ার (5V) আসছে কিনা চেক করুন
- সার্ভো কানেক্টরের **বাদামি তার = GND** নিশ্চিত করুন
- Serial Monitor এ `0,90` লিখে Motor 0 কে 90° তে নেওয়ার চেষ্টা করুন
- বাক কনভার্টারে যথেষ্ট কারেন্ট (5V/3A) আসছে কিনা দেখুন

### ❌ WiFi তে সংযোগ হচ্ছে না

- ESP32 পাওয়ার অন আছে কিনা দেখুন (OLED তে কিছু দেখাচ্ছে কিনা)
- পাসওয়ার্ড `12345678` ঠিকভাবে লিখছেন কিনা চেক করুন
- কোডে `AP_PASS` পরিবর্তন করলে মনে রাখুন **কমপক্ষে ৮ অক্ষর** হতে হবে

### ❌ রোবট ব্রাউনআউট / ক্র্যাশ করছে

- `motorCurrentDelay` বাড়ান (Settings → Motor Current Delay → 30-50 ms)
- শক্তিশালী পাওয়ার সাপ্লাই ব্যবহার করুন (5V/3A সর্বনিম্ন)
- একটি 1000µF ক্যাপাসিটর বাক কনভার্টারের আউটপুটে যোগ করতে পারেন

## 🏠 ধাপ ৭: হোম নেটওয়ার্কে সংযোগ (ঐচ্ছিক)

আপনি চাইলে রোবটকে আপনার বাসার WiFi তে সংযোগ করতে পারেন:

কোডে এই অংশ পরিবর্তন করুন:

```cpp
#define NETWORK_SSID "আপনার_WiFi_নাম"
#define NETWORK_PASS "আপনার_WiFi_পাসওয়ার্ড"
```

## 🎓 টিপস ও পরামর্শ

> [!TIP]
> - প্রথমে সার্ভো মোটর ছাড়াই কোড আপলোড ও WiFi টেস্ট করুন
> - তারপর একটি একটি করে মোটর সংযোগ ও টেস্ট করুন
> - সব কাজ করলে বডিতে অ্যাসেম্বল করুন

> [!TIP]
> - Serial Monitor (115200 baud) দিয়ে ডিবাগ করতে পারবেন
> - `0 90` লিখলে Motor 0 কে 90° তে নিয়ে যাবে
> - `all 90` লিখলে সব মোটর 90° তে যাবে
> - `run walk` বা `rn wf` লিখলে হাঁটার অ্যানিমেশন চালাবে
